Main splint.nvim API                                                  *splint*

M.linters_by_ft                                          *splint.linters_by_ft*
    A table listing which linters to run.
    The key is the filetype. The values are a list of linter names.

    Set `stop_after_first` to `true` to only run the first available linter
    in the list. When set, linters are tried in order. A linter is skipped if:
      - Its command is not executable
      - It declares `config_files` and none are found upward from the buffer
      - It declares `condition` and it returns false
    The first linter that passes all checks runs. If all are skipped, a
    warning is shown.

    Example:

    ```lua
    require("splint").linters_by_ft = {
      python = {"ruff", "mypy"},
      javascript = {"eslint", "oxlint", stop_after_first = true},
    }
    ```

    Type: ~
        (table<string,string[]>)


M.linters                                                    *splint.linters*
    Table of linter definitions. Lazy-loads built-in linters.

    Type: ~
        (table<string,splint.Linter|fun():splint.Linter>)


M.enable({opts?})                                            *splint.enable*
    Start linting. Creates autocommands and a `:Splint` command.

    ```lua
    require("splint").enable()

    -- or with custom events:
    require("splint").enable({ events = { "BufWritePost", "InsertLeave" } })
    ```

    Parameters: ~
        {opts?}  ({ events?: string[] })  Default events: BufWritePost, BufReadPost


M.disable()                                                 *splint.disable*
    Stop linting. Removes autocommands and the `:Splint` command.


                                                               *:Splint*
:Splint [name]
    Run linters on the current buffer. Without arguments, runs the
    linters configured in `linters_by_ft`. With an argument, runs the
    named linter.


                                                        *splint-checkhealth*
:checkhealth splint
    Show diagnostic information about splint configuration:
      - Whether splint is enabled and on which events
      - Which linters are configured for the current filetype
      - Whether each linter's command is executable
      - Whether config files are found (for linters that declare them)
      - Which linter would be selected when `stop_after_first` is set

    To configure diagnostics per-linter, use Neovim's diagnostic namespace:

    ```lua
    vim.diagnostic.config(
      { virtual_text = true },
      vim.api.nvim_create_namespace("eslint")
    )
    ```


splint.Linter                                              *splint.Linter*
    A Linter definition.

    Fields: ~
        {name}              (string)
        {cmd}               (string|fun(ctx:splint.Context):string)
        {args?}             ((string|fun(ctx:splint.Context):string)[])
        {stdin?}            (boolean)  send content via stdin. Defaults to false
        {append_fname?}     (boolean)  auto-append filename when stdin=false. Defaults to true
        {stream?}           ("stdout"|"stderr"|"both")  Defaults to "stdout"
        {ignore_exitcode?}  (boolean)  Defaults to true
        {env?}              (table<string,string>)
        {cwd?}              (string)
        {parser}            (splint.Parser|splint.parse)
        {config_files?}     (string[])  Known config files. Used by `stop_after_first`.
        {condition?}        (fun(ctx:splint.Context):boolean)


splint.Context                                            *splint.Context*
    Context passed to linter functions.

    Fields: ~
        {bufnr}     (integer)  buffer number
        {filename}  (string)   full path to the file
        {dirname}   (string)   directory containing the file
        {cwd}       (string)   working directory for the linter process


splint.parse                                                *splint.parse*
    Parse function for a linter.

    Type: ~
        fun(output:string,bufnr:integer,linter_cwd:string):vim.Diagnostic[]


splint.Parser                                              *splint.Parser*
    Streaming parser interface.

    Fields: ~
        {on_chunk}  (fun(chunk:string))
        {on_done}   (fun(publish:fun(diagnostics:vim.Diagnostic[]),bufnr:integer,linter_cwd:string))
